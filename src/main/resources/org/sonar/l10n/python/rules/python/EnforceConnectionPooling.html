This rule enforces connection pooling for database and HTTP network resources.

Creating raw database connections (e.g. via psycopg2.connect) or initiating raw HTTP calls (e.g. requests.get) repeatedly or inside loops is extremely expensive. It leads to socket exhaustion, high latency, and database resource starvation. Instead, use connection pooling or session objects to reuse TCP connections.

Noncompliant Code Example

import sqlite3
import requests

# Direct raw connection call
conn = sqlite3.connect("db.sqlite")  # Noncompliant

# HTTP calls inside a loop
for url in urls:
    requests.get(url)  # Noncompliant

Compliant Solution

import requests
from sqlalchemy import create_engine

# Use SQLAlchemy engine for database connections (pooled by default)
engine = create_engine("sqlite:///db.sqlite")

# Use requests.Session to reuse TCP connections across HTTP requests
with requests.Session() as session:
    for url in urls:
        session.get(url)  # Compliant
