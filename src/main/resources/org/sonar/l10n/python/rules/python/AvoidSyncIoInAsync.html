This rule detects synchronous or blocking calls (like time.sleep, open, requests.*, or urllib.request.*) inside asynchronous functions (async def).

Blocking the event loop in async Python code prevents other coroutines from running, which severely degrades performance. Use asynchronous alternatives instead (e.g., aiofiles, aiohttp, or asyncio.sleep).

Performance & Upgrade Recommendations

  • Do not starve the Event Loop: In Python async code, all coroutines run on a single thread. Calling synchronous blocking IO (like requests.get()) blocks the entire event loop, freezing all other concurrent operations.
  • Use Async Clients: Replace requests or urllib.request with httpx.AsyncClient() or aiohttp.ClientSession(). Make sure to use connection pooling correctly inside async contexts.
  • Use Non-blocking File operations: Use aiofiles instead of the standard open() function to write/read files asynchronously.
  • Replace sleeps: Replace time.sleep() with await asyncio.sleep() to yield control back to the event loop.
  • CPython 3.11+ Asyncio Improvements: Upgrading to Python 3.11+ introduces asyncio.TaskGroup (PEP 654) for safer and faster concurrent task management, and reduces asyncio runtime overhead (e.g. 2-6x faster task creation).

Noncompliant Code Example

async def fetch_data():
    time.sleep(1)  # Noncompliant
    with open("data.txt") as f:  # Noncompliant
        pass
    response = requests.get("https://example.com")  # Noncompliant

Compliant Solution

async def fetch_data():
    await asyncio.sleep(1)  # Compliant
    async with aiofiles.open("data.txt") as f:  # Compliant
        pass
    async with aiohttp.ClientSession() as session:
        async with session.get("https://example.com") as response:  # Compliant
            pass
