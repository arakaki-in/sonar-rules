This rule detects the use of list or tuple literals (including list comprehensions) as the right-hand operand of in membership tests inside loops.

When a list or tuple is used for membership testing, Python performs a linear O(n) scan. Inside a loop, this can lead to O(n*m) complexity. Converting the collection to a set gives O(1) average lookup time, dramatically improving performance for repeated membership tests.

Noncompliant Code Example

allowed = [1, 2, 3, 4, 5]
for item in data:
    if item in allowed:  # Noncompliant: O(n) lookup inside a loop
        process(item)

Compliant Solution

allowed = {1, 2, 3, 4, 5}
for item in data:
    if item in allowed:  # Compliant: O(1) lookup
        process(item)
