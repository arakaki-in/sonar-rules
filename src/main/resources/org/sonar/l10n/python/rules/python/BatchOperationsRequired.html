This rule detects single-row database modifications (like INSERT, UPDATE, DELETE, or session.add()) executed inside a loop.

Running single-row write operations in a loop creates significant overhead due to database transaction coordination, locking, and network roundtrips. Batch or bulk operation APIs (like executemany(), bulk insert APIs, or session.add_all()) should be used instead.

Noncompliant Code Example

for user in users:
    session.add(user)  # Noncompliant

Compliant Solution

session.add_all(users)  # Compliant
