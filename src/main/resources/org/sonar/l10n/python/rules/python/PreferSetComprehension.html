This rule detects when a list comprehension is passed directly to a collection constructor like set(), dict(), tuple(), or frozenset().

Passing a list comprehension to these constructors creates an unnecessary intermediate list. Python supports set comprehensions ({x for x in iterable}) and dict comprehensions ({k: v for k, v in iterable}) which build the target collection directly, avoiding the memory overhead of the intermediate list.

Noncompliant Code Example

unique = set([x.name for x in items])  # Noncompliant
tupled = tuple([x for x in data])       # Noncompliant

Compliant Solution

unique = {x.name for x in items}  # Compliant: set comprehension
tupled = tuple(x for x in data)   # Compliant: generator expression
