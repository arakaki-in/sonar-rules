This rule detects usages of threading.local and suggests using contextvars.ContextVar instead.

In modern asynchronous Python applications (e.g. using asyncio or FastAPI), multiple coroutines are executed concurrently on the same OS thread. Since threading.local is only thread-safe and not coroutine-safe, all coroutines on the same thread share the same data. This leads to severe race conditions, transaction leaks, and data corruption across concurrent requests.

Noncompliant Code Example

import threading

request_ctx = threading.local()  # Noncompliant
request_ctx.user_id = 42

Compliant Solution

import contextvars

request_ctx = contextvars.ContextVar("request_ctx", default=None)  # Compliant
request_ctx.set(42)
