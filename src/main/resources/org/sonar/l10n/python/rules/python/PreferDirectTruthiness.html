This rule detects comparisons of len(seq) with zero using operators like >, ==, !=, or <=.

Python collections (lists, dicts, sets, strings, etc.) have built-in truthiness: empty collections are falsy, non-empty collections are truthy. Using len(seq) > 0 is both less readable and slightly slower than the idiomatic if seq:. Similarly, len(seq) == 0 should be written as if not seq:.

Noncompliant Code Example

if len(items) > 0:  # Noncompliant
    process(items)

if len(items) == 0:  # Noncompliant
    return None

Compliant Solution

if items:  # Compliant
    process(items)

if not items:  # Compliant
    return None
