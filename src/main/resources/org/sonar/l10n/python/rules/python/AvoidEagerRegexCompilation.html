This rule detects calls to re.compile().

Eager regex compilation via re.compile() at module-load or instantiation time can slow down application startup. Additionally, compiling patterns inside frequently-called functions or loops is inefficient. It is recommended to use the re module functions directly (e.g. re.match(), re.search()) because they internally cache compiled regexes lazily and transparently, or to compile the regex pattern lazily only when first needed.

Noncompliant Code Example

import re

pattern = re.compile(r"\d+")  # Noncompliant

Compliant Solution

import re

# Use re functions directly; they automatically compile and cache patterns lazily
result = re.match(r"\d+", text)  # Compliant
