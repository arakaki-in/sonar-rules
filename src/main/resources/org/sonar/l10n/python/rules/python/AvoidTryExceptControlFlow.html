This rule detects when try-except blocks are used to control standard application flow, particularly for checking dictionary keys, accessing attributes, or checking indices.

Exceptions in Python are relatively expensive to construct and handle. Using try-except for standard, expected control flow (such as key presence checks) is slower than using explicit conditionals like dict.get(), in operator, hasattr(), or index bounds checking. Additionally, using exceptions for control flow makes debugging and code readability harder.

Noncompliant Code Example

try:
    value = my_dict["key"]  # Noncompliant
except KeyError:
    value = "default"

Compliant Solution

value = my_dict.get("key", "default")  # Compliant
