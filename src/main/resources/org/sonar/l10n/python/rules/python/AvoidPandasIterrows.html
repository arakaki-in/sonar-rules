This rule detects the use of pandas iterrows().

iterrows() is extremely slow because it has to create a pandas Series for each row. Vectorized operations, itertuples(), or apply() should be preferred instead.

Performance & Upgrade Recommendations

  • Avoid iterrows() on large datasets: Since iterrows() creates a pandas Series object for every single row, it introduces significant CPU and memory allocation overhead. It is typically 100x to 1000x slower than vectorized approaches.
  • Use Vectorized Operations: Always prefer built-in pandas/numpy vectorized calculations (e.g. df['C'] = df['A'] + df['B']) as they execute in optimized C-code under the hood.
  • Use itertuples() as a fallback: If you must loop over rows, use itertuples(index=False). It returns a lightweight namedtuple per row, bypassing the overhead of creating a Series object.
  • CPython 3.11+ Interpreter Speedups: Upgrading to CPython 3.11+ activates the specializing adaptive interpreter (PEP 659), which dynamically optimizes bytecode for loop executions, further accelerating iteration patterns when they cannot be completely vectorized.

Noncompliant Code Example

for index, row in df.iterrows():  # Noncompliant
    print(row['column'])

Compliant Solution

for row in df.itertuples():  # Compliant
    print(row.column)
