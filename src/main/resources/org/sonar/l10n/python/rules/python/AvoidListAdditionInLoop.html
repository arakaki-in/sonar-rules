This rule detects list concatenation using + or += [item] inside loops.

Repeated list addition inside a loop creates a new list each iteration, turning an O(n) loop into O(n^2) due to the cost of copying all existing elements. Use list.append(item) for single items or list.extend(items) for multiple items — both are O(1) amortized. This anti-pattern is a common source of severe performance degradation in Python code.

Noncompliant Code Example

result = []
for item in items:
    result = result + [process(item)]  # Noncompliant: O(n^2)

Compliant Solution

result = []
for item in items:
    result.append(process(item))  # Compliant: O(n)
