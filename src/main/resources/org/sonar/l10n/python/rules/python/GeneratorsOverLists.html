This rule detects the use of list comprehensions inside functions like sum(), any(), all(), min(), max(), or join().

List comprehensions allocate a full list in memory before passing it to the function. For large datasets, this wastes memory. Passing a generator expression (which does not have brackets []) allows the elements to be streamed and processed lazily, reducing memory allocation to $O(1)$.

Noncompliant Code Example

total = sum([item.price for item in items])  # Noncompliant

Compliant Solution

total = sum(item.price for item in items)  # Compliant
