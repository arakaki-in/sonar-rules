This rule detects in-memory aggregation of database query results (e.g., using len() or sum() on query results).

Fetching all rows from the database to aggregate them in application memory is highly inefficient and memory-intensive. Database engines are optimized for aggregation. You should use database aggregation functions (like COUNT(), SUM(), AVG()) to aggregate data directly in the query.

Noncompliant Code Example

users = session.query(User).all()
num_users = len(users)  # Noncompliant

Compliant Solution

num_users = session.query(func.count(User.id)).scalar()  # Compliant
