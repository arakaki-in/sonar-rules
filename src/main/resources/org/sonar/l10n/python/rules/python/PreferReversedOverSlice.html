This rule detects the use of the slice [::-1] on sequences to iterate or create a reversed copy.

The slice seq[::-1] creates a new reversed copy of the entire sequence, allocating O(n) memory. When the goal is to iterate in reverse order, the built-in reversed(seq) returns an iterator that traverses the sequence backwards without copying it, providing O(1) memory overhead and often better performance.

Noncompliant Code Example

for item in seq[::-1]:  # Noncompliant: creates a full copy
    process(item)

Compliant Solution

for item in reversed(seq):  # Compliant: O(1) memory, no copy
    process(item)
