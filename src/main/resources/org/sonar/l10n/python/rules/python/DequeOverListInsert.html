This rule detects when insert(0, ...) or pop(0) are called on list-like objects.

Inserting or popping at the beginning of a Python list is O(N) because all subsequent elements must be shifted in memory. Using collections.deque provides O(1) performance for these operations.

Performance & Upgrade Recommendations

  • Time Complexity Penalty: A standard python list is a dynamic array. Inserting/popping at index 0 shifts all elements in memory, yielding a time complexity of O(N). Doing this in a loop results in O(N^2) quadratic complexity.
  • Double-Ended Queue (Deque): collections.deque is implemented as a doubly linked list, enabling O(1) constant-time push and pop operations at both the beginning and the end.
  • Upgrade queue logic: For FIFO queues, stacks, or rotating lists, replace lists with collections.deque and use appendleft() / popleft().
  • CPython 3.11+ Interpreter Speedups: Under CPython 3.11+, bytecode operations are dynamically optimized by the specializing adaptive interpreter (PEP 659), enhancing the execution speed of collections.deque methods.

Noncompliant Code Example

my_list = []
my_list.insert(0, "item")  # Noncompliant
my_list.pop(0)  # Noncompliant

Compliant Solution

from collections import deque
my_queue = deque()
my_queue.appendleft("item")  # Compliant
my_queue.popleft()  # Compliant
