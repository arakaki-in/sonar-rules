This rule checks for comparisons with None using == or !=.

According to PEP 8, comparisons to singletons like None should always be done with is or is not, never the equality operators. Equality operators can be customized by classes (via __eq__), which might lead to unexpected behavior or performance issues.

Noncompliant Code Example

if val == None:  # Noncompliant
    pass
if val != None:  # Noncompliant
    pass

Compliant Solution

if val is None:  # Compliant
    pass
if val is not None:  # Compliant
    pass
