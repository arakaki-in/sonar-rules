This rule detects when database queries are executed inside a loop, causing the N+1 query problem.

Executing queries inside a loop leads to severe performance degradation because it incurs network latency and database overhead for every iterated item. Instead, retrieve all required data in a single batch query (e.g. using SQL joins, prefetching, or bulk queries) before entering the loop.

Performance & Upgrade Recommendations

  • Database Round-Trips: Executing a SQL query inside a loop executes N queries for N elements (N+1 query problem). This amplifies network latency and resource consumption exponentially.
  • Eager Loading with Joins: In SQLAlchemy, use options(joinedload(Model.relationship)) to load related objects in a single query via SQL JOINs.
  • Batch Fetching: Use IN-clauses (e.g. Model.id.in_(ids)) to retrieve all records in a single query before performing iterations.
  • CPython 3.11+ Interpreter Speedups: Running under CPython 3.11+ leverages adaptive specialization (PEP 659) to significantly reduce the overhead of Python-level loops, object instantiations, and attribute accesses when fetching query results.

Noncompliant Code Example

for user_id in user_ids:
    user = session.query(User).filter(User.id == user_id).first()  # Noncompliant
    print(user.name)

Compliant Solution

users = session.query(User).filter(User.id.in_(user_ids)).all()  # Compliant
for user in users:
    print(user.name)
