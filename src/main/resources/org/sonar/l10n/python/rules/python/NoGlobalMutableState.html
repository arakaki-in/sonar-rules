This rule detects global mutable state by flagging the use of the global keyword and the declaration of mutable objects (like lists, dicts, sets, or custom class instances) at the module level.

Global mutable state introduces severe thread-safety and concurrency hazards, leading to race conditions, unpredictable behaviors, and memory leaks when multiple threads or coroutines run concurrently.

Noncompliant Code Example

shared_list = []  # Noncompliant

def add_item(item):
    global counter  # Noncompliant
    shared_list.append(item)

Compliant Solution

import contextvars

# Use thread-safe or task-safe context variables instead of global mutable objects
shared_items = contextvars.ContextVar("shared_items", default=tuple())

def add_item(item):
    current = shared_items.get()
    shared_items.set(current + (item,))
