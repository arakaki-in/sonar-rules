This rule detects the use of for i in range(len(seq)) pattern for iterating over sequences.

This pattern is both less readable and slower than Python's idiomatic alternatives. Direct iteration (for item in seq) avoids index lookups entirely. When the index is needed, enumerate(seq) provides both the index and value efficiently. For iterating over multiple sequences in parallel, zip() is cleaner and faster than indexing with range(len()).

Noncompliant Code Example

for i in range(len(items)):  # Noncompliant
    process(items[i])

Compliant Solution

for item in items:  # Compliant: direct iteration
    process(item)

for i, item in enumerate(items):  # Compliant: when index is needed
    process(i, item)
