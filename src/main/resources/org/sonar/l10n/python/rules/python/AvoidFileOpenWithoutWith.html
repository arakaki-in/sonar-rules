This rule detects calls to the built-in open() or io.open() functions that are not managed by a with context manager statement. Opening files without using a with statement can lead to unclosed file handles and resource leaks if an exception occurs before the file is explicitly closed.

Noncompliant Code Example

f = open("file.txt", "w")
f.write("Hello")
f.close()

Compliant Solution

with open("file.txt", "w") as f:
    f.write("Hello")
